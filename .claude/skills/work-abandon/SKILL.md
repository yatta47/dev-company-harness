「これはやめる」「調べたけどやらないでいい」で呼ばれる。**理由は必須**。

```bash
python3 -m harness.cli abandon --reason "<理由>"
```

理由は `dod.json` の `abandon_reason` に書き込まれ、状態は `abandoned`（終端）になる。任意の非終端状態から呼べるが、既に `approved` / `abandoned` なら弾かれる。

## 理由を書かせる

`$ARGUMENTS` に理由がない、または「不要になった」程度しかない場合は、AskUserQuestion で**判断の中身**を聞いてから実行する。打ち切りの価値は理由にしかない。

書くべきこと:
- 何がわかって、やめる判断になったのか（調査で判明した事実があるなら具体的に）
- どういう条件が変われば再検討する価値があるか

**来期に誰かが同じことを提案したとき、この1行が読まれる。** それを想定して書く。

## 打ち切る前に確認する

- 一時的に止めたいだけなら打ち切らない。タスクは放置しても消えないので、そのまま別タスクへ `/work-switch` すればよい。
- 「方針が違うだけ」なら打ち切りではなく差し戻し（→ `/work-return`）。
- **打ち切りは取り消せない**（終端状態）。実行前に理由をユーザーに提示して確認を取る。

## 役割境界
- **する**: タスクの打ち切りと理由の記録のみ。
- **しない**: 前工程への差し戻し（→ `/work-return`）、対象repoの作業ツリーの後始末（人間が行う）。
